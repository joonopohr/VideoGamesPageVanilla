Features of Game Engines

Defintions

  1. Scene: Sections of a game that includes objects within that section. A scene could be a level in a game or the menu for a game.
  2. Object: A things that can be seen in a scene, it could be something like a player model or a tree.
  3. Input: Information that is received by the computer when an action is taken such as when a person clicks a mouse or enters the upkey.
  4. Assets: Preset tools that are used in game development to simplify the game development process.

What is it

What are some of the features in game engines?

Game engines includes many things such as a rendering engine, resource libaries and managers for scenes, memory, inputs, audio. During this section, we will further look into what each feature cando in creating a game.

What do each feature do?
Scene Manger
This manages the objects2 and resources within a scene1 and makes sure that they are placed in the correct postion.
Rendering Engine
It is an engine that helps with the process of creating a 2D / 3D scene1 for the player to see through the use of a file that store multiple details such as where the object is placed, where the lighting is from and the height and width of the scene1.
Resource libraries
A list of assets4 that are intergrated within the game engine.
Inside the libary, there are many things such as an objects2 ranging from a chair to book or a pre-built menu screen.
Scripting
A feature that allows the developer to add their own code to do things that the game engine can't do by itself.
Scripting for example can allow the player to interact with an object.
Memory Manager
Memory Manager is used to allocate memory to functions that will be used within the game and can also unallocate once the functionis no longer needed.
This is useful as having every function run at the same time would overwhelm the computer.
Input Manager
The input management deals with what happens when a player puts an input3 into the game.
What the input management does is receive input3s from either keyboards, mouses, or joysticks, and with that information, the game is told about the received input3 and then the game does whatever is required when a player presses an input3.
Audio Manager
This feature helps with determining when audio outputs are being used within the game.
